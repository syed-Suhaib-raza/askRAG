# askRAG

askRAG is an **offline Retrieval-Augmented Generation (RAG) agent for Wikipedia** designed to run **entirely on-device**.
It allows users to query a **local Wikipedia dump** and receive answers generated by a local LLM using a hierarchical retrieval pipeline.

The system is optimized for **efficiency, modularity, and privacy**, enabling powerful knowledge retrieval without relying on external APIs or internet connectivity.

---

# Features

* **Fully Offline**

  * Works with downloaded Wikipedia dumps (without media)
  * No internet required after setup

* **Hierarchical Retrieval Pipeline**

  * Page-level retrieval
  * Section-level retrieval
  * Chunk-level retrieval

* **Hybrid Search**

  * Vector search using FAISS
  * BM25 lexical retrieval
  * Optional graph retrieval

* **Local LLM Generation**

  * Works with locally hosted models (e.g., Ollama, Mistral)

* **Context Optimization**

  * Query rewriting
  * Context compression
  * Conversation memory

* **Streamlit Interface**

  * Simple UI for interacting with the system

# Wikipedia Data

askRAG works with an **offline Wikipedia dump without media files**.

Place the downloaded dump inside:

```
data/wikipedia_dump/
```

The system processes the dump to extract articles, sections, and chunks for retrieval.

---

# Ingestion Pipeline

The ingestion pipeline converts the raw Wikipedia dump into structured data.

```
python scripts/run_ingestion_pipeline.py
```

The pipeline performs:

1. Extract Wikipedia pages
2. Clean and filter sections
3. Chunk text
4. Generate embeddings
5. Deduplicate chunks

---

# Index Building

After ingestion, retrieval indexes must be built.

```
python scripts/rebuild_indexes.py
```

This generates:

* Page index
* Section index
* Chunk index
* BM25 index
* FAISS vector index

These indexes are stored in:

```
processed_data/
```

---

# Running the Application

Launch the Streamlit interface:

```
streamlit run app/main.py
```

This opens a browser interface where you can query the system.

---

# Retrieval Pipeline

askRAG uses a **hierarchical retrieval approach**:

1. Retrieve relevant **pages**
2. Retrieve relevant **sections** from those pages
3. Retrieve relevant **chunks**
4. Compress and rank context
5. Send context to the LLM for answer generation

This approach improves both **accuracy and retrieval efficiency**.

# Example Query

```
What is the theory of relativity?
```

The system will:

1. Retrieve relevant Wikipedia pages
2. Extract the most relevant sections and chunks
3. Provide the context to the LLM
4. Generate an answer
